# 数独挑战 - 1000关

## 项目结构
- `sudoku.html` — 单文件完整应用（HTML + CSS + JavaScript）

## 架构设计

### 谜题生成（种子驱动）
- **seed = level × 137 + 42** 生成完整数独棋盘
- **seed = level × 7919 + 314159** 控制挖空顺序
- 相同关卡号永远产生相同谜题

### 棋盘生成流程
1. 从 `BASE_GRID`（一个已知有效解）开始
2. 依次应用：数字置换 → 行排列 → 列排列 → 带排列 → 栈排列 → 再次行/列排列
3. 得到随机化但始终有效的完整棋盘

### 挖空算法
1. 基于种子打乱 81 个位置
2. 依次尝试移除数字，每次用 `countSolutions(grid, 2)` 验证唯一解
3. 唯一解则保留挖空，否则恢复
4. 目标挖空数：30 + floor(t^0.82 × 26)，t = (level-1)/999

### 难度分布
| 关卡 | 难度 | 给定数字 |
|------|------|---------|
| 1-200 | 简单 | ~45-49 |
| 201-450 | 中等 | ~37-44 |
| 451-700 | 困难 | ~30-36 |
| 701-1000 | 专家 | ~25-29 |

### 核心技术
- **求解器**：回溯 + 最少候选优化（MRV），计数解时找到 2 个即停
- **UI**：CSS Grid 布局，fixed 定位浮动选择器
- **持久化**：localStorage（每 10 秒自动存档 + 页面关闭前保存）

## 关键功能
- 1000 个独立关卡（1-1000 输入或 ◀▶ 导航）
- 浮动数字选择器（点击格子弹出 1-9）
- 底部数字小键盘
- 笔记模式（N 键切换，标记候选数）
- 撤销（Ctrl+Z，最多 50 步）
- 提示（自动找最受约束的空格）
- 错误检查（红色闪烁）
- 计时器 + 进度条
- 通关弹窗 + 自动存档

## 键盘快捷键
| 键 | 功能 |
|----|------|
| 1-9 | 输入数字 |
| ↑↓←→ | 移动选中格 |
| Delete/Backspace | 擦除 |
| N | 笔记模式 |
| Ctrl+Z | 撤销 |
| Enter（在关卡输入框） | 跳转关卡 |

## 已知问题和待优化
- 关卡 700+ 高难度时生成可能略慢（~500ms），可考虑加加载动画
- 浮动选择器在极端小屏（<360px）可能布局偏挤
- 可增加胜利音效或振动反馈
- 提示功能可增加逐步提示（先显示候选数再填答案）
- 可增加计时排行榜（本地存储）
- 笔记模式下浮动选择器可加「N」标识以示区别
